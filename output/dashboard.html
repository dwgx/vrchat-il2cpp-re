🔒 EAC / 认证分析

四令牌认证链

令牌 1：Steam App Ticket 来源：SteamAPI_ISteamUser_GetAuthSessionTicket 使用方：VRChat API（api.vrchat.cloud） 传输：HTTPS POST 状态：无 EAC 也可正常工作 | v 令牌 2：VRChat API Token 来源：VRChat API 登录响应（JWT） 使用方：Photon Custom Auth 传输：Photon OpCode 231 状态：无 EAC 也可正常工作 | v 令牌 3：Photon Auth Token 来源：PlatformOptions.GetKeyServerAuthToken（RVA 0x112A080） 使用方：Photon Name Server 传输：OpCode 231（AuthenticateOnce） 状态：无 EAC 也可正常工作 | v 令牌 4：EAC Session Token 来源：EOS SDK AddNotifyMessageToServer 回调 使用方：Photon 服务端插件 / VRChat 后端 状态：绕过模式下缺失，导致加入房间被拒

服务端校验时间线

[+0.0s] 生成 Steam ticket（仅客户端本地） [+0.2s] VRChat API 登录（HTTPS）-- 令牌 1 --> 令牌 2 [正常] [+0.3s] EOS_AntiCheatClient_BeginSession [绕过下未调用] [+1-3s] AddNotifyMessageToServer 回调触发 [绕过下永不触发] [+5s] Photon Connect（AuthenticateOnce，OpCode 231） 发送令牌 2 + 令牌 3（由 GetKeyServerAuthToken 组装） 服务端 webhook 对接 VRChat API 进行校验 [正常] [+22s] OnCustomAuthenticationResponse -- Photon 认证成功 [正常] [+22s] OnConnectedToMaster [+23s] JoinRoom（OpCode 226） 服务端检查：auth? 是 EAC session? 否（绕过） --> 空错误拒绝：Failed to join...due to ''

EOS 反作弊协议

客户端 服务端 | | |-- BeginSession ------------------> |-- RegisterClient | | （开始超时计时） | | |<-- opaque message（完整性）------ |-- AddNotifyMessageToClient |-- ReceiveMsg | | | |-- opaque message（响应）---------> |-- ReceiveMessageFromClient | （AddNotifyMessageToServer） | （校验完整性） | | | ... 持续交换消息 ... | | | | （如果客户端停止发送） |-- AuthenticationFailed | |-- ClientActionRequired: REMOVE

这些消息由 EOS SDK 在内部进行不透明、加密、签名处理，无法伪造。
