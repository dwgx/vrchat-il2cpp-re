What it does

A pipeline of Python utilities that read MDMP minidumps and Windows complete kernel dumps, walk the live Il2CppClass structures inside, and output class / method / field metadata in a form a developer can browse offline.

Designed for the case where a Unity title uses Beebyte string obfuscation: the tools recover layout (offsets, types, parents) without relying on readable name strings, then optionally cross-map two dumps taken across a build update so that prior vocabulary survives a re-seed.

Pipeline

Windows Complete Memory Dump (MEMORY.DMP) — produced by the user ↓ kerneldump_to_minidump.py extract one process to MDMP via Volatility 3 ↓ extract_precise_dump.py --auto-heap scan all user-mode regions for Il2CppClass ↓ extract_field_types_from_dump.py resolve (name, type, offset) per FieldInfo ↓ cross_version_class_map.py method-body hash + name-sig matching ↓ merge_field_types.py apply types to deobfuscated dump ↓ generate_source_tree.py namespace-organised C# source tree

Requirements

pip install volatility3 frida psutil

Tested on Windows 11 (build 26200), Python 3.12, Volatility 3 2.27, Frida 17.7. Targets MDMP minidumps containing the standard MEMORY_LIST_STREAM_64 stream.

Out of scope

  • No process injection, no in-memory patching of running games.
  • No anti-cheat circumvention.
  • No redistribution of game executables, libraries, or assets.
  • No publication of obfuscation-name dictionaries belonging to any specific title.
